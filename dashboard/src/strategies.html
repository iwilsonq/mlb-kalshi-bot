Pitcher Strikeouts (pitcher_ks)

Active

Predicts the number of strikeouts a starting pitcher will record using a Poisson model. Trades both YES (pitcher exceeds threshold) and NO (market overprices the line).

Signal Pipeline

1. Fetch pitcher profile → 2. Compute λ (expected Ks) → 3. P(≥n) via Poisson CDF → 4. Calibrate (isotonic regression) → 5. Edge = calibrated% − market price → 6. Kelly sizing

Lambda Formula

// Base: blend recent form (70%) with season rate (30%) base = 0.70 × recent_k_per_start + 0.30 × (k_per_9 / 9 × recent_ip) // Adjustments (all dampened toward 1.0) opp_adj = 1.0 + 0.50 × (opp_k_rate / 0.225 − 1.0) whiff_adj = 1.0 + 0.50 × (whiff_rate / 0.245 − 1.0) chase_adj = 1.0 + 0.30 × (chase_rate / 0.285 − 1.0) velo_adj = 1.0 + 0.01 × (fastball_velo − 93.5) // Ceiling: cap at max Ks observed this season + 1 lam = min(base × opp × whiff × chase × velo, max_k_in_start + 1) // Deflation: model over-predicts by ~15-20% lam ×= 0.85

Player Hits (player_hits)

Active

Predicts the number of hits a batter will record using a Poisson model with Bayesian shrinkage, Statcast quality-of-contact data, and recent form blending.

Lambda Formula

// Bayesian-shrunk batting average (prior: .243 over 250 AB) eff_avg = (split_hits + 0.243 × 250) / (split_ab + 250) // Three-way blend: season (40%) + Statcast xBA (30%) + recent 7-game avg (30%) blended = 0.40 × eff_avg + 0.30 × xBA + 0.30 × recent_avg // Base lambda lam = blended × expected_ab(batting_order) // Adjustments pitcher_adj = min(1.0 + 0.50 × (opp_whip / 1.28 − 1.0), 1.35) hhr_adj = 1.0 + 0.25 × (hard_hit_rate / 0.370 − 1.0) park = HIT_PARK_FACTORS[home_team] // 0.93 – 1.12 lam ×= pitcher_adj × hhr_adj × park // Deflation: model over-predicts hits by ~2:1 lam ×= 0.80

Player Home Runs (player_hr)

Disabled

Predicts single-game home run probability using a Poisson model with Bayesian shrinkage, Statcast power metrics, and recent HR form. Currently disabled due to 0% historical calibrated win rate across 456 samples.

Lambda Formula

// Bayesian-shrunk HR rate (prior: .017 HR/AB over 300 AB) eff_rate = (split_hr + 0.017 × 300) / (split_ab + 300) // Blend with recent HR rate (last 7 games, 30% weight) recent_rate = recent_hr / (7 × 3.9) eff_rate = 0.70 × eff_rate + 0.30 × recent_rate lam = eff_rate × expected_ab(batting_order) // Adjustment factors (collected, then capped as a group) adj = pitcher_hr9_adj // min(opp_hr9 / 1.1, 1.5) adj ×= park_factor // 0.82 (SF) – 1.38 (COL) adj ×= barrel_adj // 1 + 0.5 × (barrel / 0.065 − 1) adj ×= ev_adj // 1 + 0.03 × (exit_velo − 88.5) adj ×= xslg_adj // 1 + 0.3 × (xSLG / 0.400 − 1) adj ×= pitcher_bra_adj // 1 + 0.3 × (bra / 0.065 − 1) // CAP total adjustment to prevent multiplicative blowup adj = min(adj, 1.50) lam ×= adj // Deflation: model over-predicts HR by ~2-3x lam ×= 0.60

Game Winner (game_winner)

Disabled

Predicts home/away win probabilities using a multi-factor log5-inspired model. Disabled due to −206% ROI — the ERA-based heuristic lacks the precision needed to find edge against efficient moneyline markets.

Rating Formula

rating = 0.40 × pitching_quality // league_avg_ERA / pitcher_ERA + 0.40 × offense_quality // team_runs_per_game / 4.50 + 0.10 × bullpen_quality // league_avg_ERA / bullpen_ERA + 0.10 × record_quality // team_win_pct / 0.500 home_prob = (home_rating × 0.540) / (home_rating × 0.540 + away_rating × 0.460) // Clamped to [30%, 70%]
Only 3 calibration samples. The model uses crude ERA-based pitcher quality without Statcast adjustments, and the bullpen ERA field is never actually populated from the API.

Shared Infrastructure

Calibration Layer

Isotonic regression (pool adjacent violators algorithm) fitted on 5%-wide probability bins. Maps raw model probabilities to calibrated probabilities based on historical signal outcomes.
